2.5 Micronaut Transactional Advice

Since 2.0, by default Micronaut-based transaction management is enabled. Thus on any service you can use the io.micronaut.transaction.annotation.TransactionalAdvice annotation.

In the previous service:

2.5.1 Java EE 7 javax.transaction.Transactional

Another option is to use Java EE 7 javax.transaction.Transactional annotation to mark your transaction demarcations.

But to use it, you have to include the micronaut-data-processor dependency in your annotation processor configuration:

build.gradle
...
..
dependencies {
...
..
.
    annotationProcessor "io.micronaut.data:micronaut-data-processor"
    implementation("io.micronaut.data:micronaut-data-hibernate-jpa")
}

Then, you will use javax.transaction.Transactional:

src/main/java/example/micronaut/GenreRepositoryImpl.java
import javax.transaction.Transactional;
...
..
.
@Singleton
public class GenreRepositoryImpl implements GenreRepository {
...
..
.
    @Transactional
    public Optional<Genre> findById(@NotNull Long id) {
...
..
.
    @Transactional
    public Genre save(@NotBlank String name) {
...
..
.
    @Transactional
    public void deleteById(@NotNull Long id) {
...
..
.
    @Transactional
    public List<Genre> findAll(@NotNull SortingAndOrderArguments args) {
...
..
.
    @Transactional
    public int update(@NotNull Long id, @NotBlank String name) {
...
..
.
    @Transactional
    public Genre saveWithException(@NotBlank String name) {
